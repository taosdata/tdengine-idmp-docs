# 9.6 相关分析

相关分析是工业数据分析中用于量化变量间统计依赖关系的核心方法。IDMP 支持对时序数据执行相关性分析，帮助用户发现设备参数之间、工艺变量之间或跨系统指标之间的内在联动规律，为特征筛选、根因溯源和控制优化提供定量依据。

## 相关分析原理

相关分析的核心问题是：当一个变量发生变化时，另一个变量是否表现出与之协同变化的趋势，以及这种协同变化的方向（正相关/负相关）和强度如何。需要注意的是，相关分析本身不能证明因果关系，但能有效缩小分析范围、验证物理假设，并为后续建模提供定量依据。

在工业场景中，相关分析通常服务于以下目标：

- **影响因素识别：** 在众多候选变量中快速筛选出与目标指标显著相关的关键因素
- **物理假设验证：** 将工程经验或工艺知识转化为可量化的假设，通过数据加以验证
- **隐性关联发现：** 识别表面无关但实际存在统计依赖的变量对，为跨系统协同优化提供线索

准确的相关性分析对数据质量有一定要求：参与分析的两列数据应覆盖相同时间范围且采样频率一致；数据量应足够充分（通常不少于 30 个有效数据点）；对于含有明显趋势或季节性成分的数据，建议先进行去趋势处理，以避免共同趋势引起的伪相关（Spurious Correlation）。

此外，皮尔森相关系数仅反映线性关系。若变量之间存在时间滞后（如上游参数变化需经过一定时间才影响下游指标），同步相关系数会低估真实关联强度，此时应使用 `TLCC` 进行时延相关分析。

## 支持的分析方法

IDMP 通过 TDengine 内置的相关分析函数提供以下三类分析能力：

| 方法 | 函数 | 特点 |
|---|---|---|
| **皮尔森相关系数** | `CORR` | 度量两个变量之间线性关系的方向和强度，取值范围 [−1, 1]；+1 表示完全正相关，−1 表示完全负相关，0 表示无线性相关；计算效率高，结果可解释性强（默认方法，无需 TDgpt） |
| **动态时间规整** | `DTW` | 通过动态规划对两条时序进行非线性时域对准后计算相似度；适用于存在时间拉伸、相位偏移或采样率不一致的时序对比；返回值越小表示相似度越高（需要 TDgpt） |
| **时延互相关** | `TLCC` | 计算两个时序在不同时间滞后（lag）下的线性相关系数序列，用于识别一个变量的变化对另一个变量产生延迟影响的方向、强度及估算传导时延（需要 TDgpt） |

### 方法选择建议

- 对于同步采集的两个指标之间的线性关联分析，优先使用 `CORR`，结果直观，可直接量化关联强度
- 对于形态相似但存在时间偏移的两条时序，或采样频率不一致的对比场景，选择 `DTW`
- 对于分析因果传导链中的延迟效应（如上游参数变化对下游指标的滞后影响），选择 `TLCC`

## 使用入口

当前版本通过 TDengine SQL 函数调用相关分析能力，可在 IDMP 的数据查询与探索界面中执行。

### CORR 函数

`CORR` 函数计算两列时序数据的皮尔森相关系数，不依赖 TDgpt，可在任意 TDengine 环境中直接使用。

```sql
-- 计算两个属性在指定时间范围内的皮尔森相关系数
SELECT CORR(temperature, power_consumption)
FROM equipment_metrics
WHERE ts >= '2024-01-01' AND ts < '2024-04-01';

-- 按设备分组，分别计算每台设备的相关系数
SELECT device_id, CORR(vibration, bearing_temp)
FROM motor_sensors
PARTITION BY device_id;

-- 按滚动时间窗口计算动态相关系数
SELECT _wstart, CORR(flow_rate, pressure)
FROM pipeline_data
INTERVAL(1h);
```

:::note
`CORR` 函数要求输入至少 2 个有效数据点，否则返回 NULL。函数可与 `PARTITION BY`、`INTERVAL` 等子句配合使用，支持分组和窗口化的相关性计算。
:::

### DTW 函数

`DTW` 函数通过动态规划对两条时序进行非线性时域对准，计算其相似度（曼哈顿距离之和），返回值越小表示两序列越相似。

```sql
-- 计算两条时序的动态时间规整相似度（默认邻域半径）
SELECT DTW(col1, col2) FROM foo;

-- 指定邻域搜索半径为 2，提升精度（计算开销随之增加）
SELECT DTW(col1, col2, 'radius=2') FROM foo;
```

`DTW` 最大支持 10240 行数据输入；`radius` 参数控制路径搜索的邻域半径（默认值为 1），较大的 `radius` 精度更高但计算成本更大。

### TLCC 函数

`TLCC` 函数计算两列时序在不同时间滞后步数下的线性相关系数，返回各滞后步数对应的相关系数序列，用于识别变量间的传导时延。

```sql
-- 在默认滞后范围 [−1, 1] 内计算时延相关性
SELECT TLCC(col1, col2) FROM foo;

-- 指定滞后范围为 [−12, 0]，分析 col1 对 col2 的延迟影响
SELECT TLCC(col1, col2, 'lag_start=-12, lag_end=0') FROM foo;
```

结果中相关系数绝对值最大处对应的 lag 值，即为估算的物理传导时延步数。

### 未来规划

后续版本将在 IDMP 可视化界面中提供相关分析的图形化操作入口，规划中的功能包括：

- 选择多个属性变量，一键生成两两相关系数的**热力图（Correlation Heatmap）**，直观呈现多变量间的关联矩阵
- 在趋势图或散点图面板中直接进行相关分析，结果叠加显示在当前图表上
- 按滑动时间窗口动态追踪相关性变化，实时监测变量间关联结构的演变

## 应用场景

相关分析在工业领域具有广泛的落地价值：

**能源与电力**

- 计算电力负荷与气温、日照时长等气象指标的相关系数，为负荷预测模型筛选关键输入特征
- 对同一风场多台风机的功率序列进行两两相关分析，识别与整体趋势异常偏离的风机

**设备工况监测**

- 动态追踪旋转设备振动、温度、电流信号之间的相关性，当原本高度相关的信号突然解耦时，往往预示设备状态异常
- 对上游工艺参数与下游质量指标进行时延相关分析，定位物理传导时延，辅助闭环控制策略优化

**生产制造**

- 计算多个工艺参数（注射温度、保压时间、冷却速率等）与产品良品率的相关系数，快速识别关键质量控制变量
- 对多条生产线的节拍信号进行相似度分析，识别偏离标准的产线，辅助工艺对标

**流程工业**

- 分析原料成分含量与产品品质指标的相关关系，为配方调整和原料筛选提供定量依据
- 分析蒸汽、循环水等公用工程消耗量与主工艺负荷的相关性，辅助能源调度和容量规划

**环境与公用事业**

- 计算各支管水质指标与总出水水质的相关系数，辅助污水水质溯源排查
- 分析建筑能耗与室外气象参数的相关性，量化气候因子对冷热负荷的贡献，为动态能耗基准建立提供依据

### 示例：验证室外温度与楼宇中央空调能耗的关联强度

**场景背景**

某商业楼宇的能源管理团队发现每年夏季电力消耗显著增加，初步判断与室外高温导致空调负荷上升有关，但缺乏定量依据。团队希望通过数据验证这一假设，量化温度对空调能耗的影响程度，为动态能耗基准的制定提供参考。

**操作过程**

在 IDMP 的数据查询界面，对全年逐小时数据执行以下查询：

```sql
SELECT CORR(outdoor_temp, hvac_power)
FROM building_energy
WHERE ts >= '2024-01-01' AND ts < '2025-01-01';
```

**分析效果**

查询返回 `0.87`，表明室外温度与中央空调能耗之间存在显著正相关关系——室外温度越高，空调系统能耗越大，且关联强度在全年范围内保持稳定。

能源管理团队以此为依据，建立了以室外温度为分段变量的动态能耗基准：在不同温度区间下分别设定中央空调系统的正常能耗参考范围。当实际能耗偏离对应温度段的基准超过 15% 时，自动触发能效异常告警。上线后首季度，团队通过告警发现了 2 次冷机效率下降事件，均在问题扩大前得到及时处置。
