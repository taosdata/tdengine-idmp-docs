# 7.2 创建分析

要手工创建新分析，请导航到元素的**分析**标签页并点击工具栏中的 **+**，或点击空列表中的 **+ 手工创建新分析**按钮。这将打开分析创建表单。

表单分为四个按顺序填写的编号区段：**基本信息**、**触发**、**计算**和**事件**。

## 7.2.1 基本信息

| 字段 | 说明 |
|---|---|
| **名称**（必填） | 此分析的唯一名称。保持简洁且具有描述性——例如"每小时最大电压"或"压缩机效率"。 |
| **类别** | 用于组织和过滤分析的可选类别标签。可以内联创建新标签。 |
| **创建后立即启用分析** | 勾选时（默认），分析在保存后立即开始运行。取消勾选可创建处于暂停状态的分析。 |
| **对乱序数据重新计算** | 勾选时，如果数据迟到（时间戳早于已处理数据），分析将对受影响的窗口重新运行。适用于传感器数据可能延迟或乱序到达的场景。 |
| **描述** | 此分析计算内容和原因的可选自由文本描述。 |

## 7.2.2 触发

触发区段定义分析何时运行。所有八种触发类型及其参数的完整说明请参见[触发类型](./03-trigger-types.md)。

所有触发类型共享两个可选的公共字段：

- **预过滤** — 在触发评估前应用于数据的过滤表达式。只有满足过滤条件的数据行才会被考虑。适用于忽略无效读数（例如在计算平均值前过滤掉零值）。
- **重算历史** — 启用时，分析对历史数据运行以回填计算输出。启用此字段后会显示两个附加选项：
  - **优先重算历史** — 勾选时，分析在开始处理新实时数据之前先处理所有历史数据。
  - **开始时间** — 历史数据回填的起始日期和时间。

## 7.2.3 计算

计算区段定义分析计算什么以及结果存储在哪里。完整说明请参见[计算](./04-calculation.md)。

## 7.2.4 事件

事件区段控制分析每次触发时是否生成事件。此区段默认禁用。勾选**生成**复选框以启用。完整说明请参见[生成事件](./05-generating-events.md)。

## 7.2.5 保存和放弃

点击**保存**创建分析。若勾选了**创建后立即启用分析**，分析立即开始运行，并以**运行中**状态出现在列表中。

点击**放弃**取消。若有未保存的更改，会显示确认对话框。

:::tip
表单右侧包含一个可折叠的**用户指南**面板，在填写各字段时提供说明。
:::
