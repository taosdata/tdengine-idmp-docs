# 7.5 生成事件

事件区段（分析表单第 4 段）控制分析每次触发时是否创建事件。事件生成默认禁用。

## 7.5.1 启用事件生成

在事件区段标题中勾选**生成**复选框以启用事件生成。启用后，事件字段变为可用，第 3 段的输出属性表格中会出现**事件属性**列。

禁用事件生成时，该区段显示"未启用事件生成"，不产生任何事件。

## 7.5.2 事件字段

启用事件生成后，需要配置以下字段以定义事件的结构、严重程度和发送策略。

| 字段 | 说明 |
|---|---|
| **模板**（必填） | 定义事件结构（包括名称、属性和显示设置）的事件模板。可从下拉菜单中选择现有模板，或点击 **+ 新建事件模板**进行内联创建。 |
| **允许确认** | 勾选时（默认），操作人员可对该分析生成的事件进行确认操作。 |
| **严重程度**（必填） | 生成事件的严重等级。选项通常包括警告、严重及系统中定义的其他级别，默认为警告。 |
| **原因代码** | 用于对事件原因进行分类的可选枚举值。原因代码选项在**基础库 → 枚举集**中定义。 |
| **原因代码值** | 所选原因代码中的可选子值，选择原因代码后可用。 |
| **最小发送间隔** | 该分析产生的连续事件之间的最小时间间隔。若在此间隔结束前再次触发，则不生成新事件。默认值为 15 分钟，用于防止条件持续触发时事件泛滥。 |

## 7.5.3 在事件中捕获计算值

启用事件生成后，**事件属性**列在输出属性表格（第 3 段）中变为可用。该功能允许在事件创建时刻将分析表达式的计算值直接捕获到事件属性中。

例如，若分析计算窗口期间的最大电压，可将该值同时写入元素属性（用于时序历史记录）和事件属性（随事件一同记录）。每个输出行可独立配置为写入元素属性、事件属性或同时写入两者。

## 7.5.4 与事件模板的关系

此处选择的事件模板决定以下内容：

- 事件名称和显示标签
- **事件属性**列中可选择的事件属性
- 该分析生成的事件在事件标签页中的呈现方式

分析生成的事件与手动创建的事件一同显示在元素的事件标签页中，并以分析名称作为来源进行标注。
