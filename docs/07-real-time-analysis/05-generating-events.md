事件区段（分析表单第 4 段）控制分析每次触发时是否创建事件。事件生成默认禁用。

## 启用事件生成

在事件区段标题中勾选**生成**复选框以启用事件生成。启用后，事件字段变为可用，第 3 段的输出属性表格中会出现**事件属性**列。

禁用事件生成时，该区段显示"未启用事件生成"，不产生任何事件。

## 事件字段

| 字段 | 说明 |
|---|---|
| **模板**（必填） | 定义事件结构（包括名称、属性和显示设置）的事件模板。从下拉菜单中选择现有模板，或点击 **+ 新建事件模板**以内联方式创建。 |
| **允许确认** | 勾选时（默认），操作人员可以确认此分析生成的事件。 |
| **严重程度**（必填） | 生成事件的严重等级。选项通常包括警告、严重以及系统中定义的其他级别。默认为警告。 |
| **原因代码** | 对事件原因进行分类的可选枚举值。原因代码选项在**基础库 → 枚举集**中定义。 |
| **原因代码值** | 所选原因代码中的可选子值。选择原因代码后可用。 |
| **最小发送间隔** | 此分析产生的连续事件之间的最小时间间隔。若触发在此间隔结束前再次发生，则不生成新事件。默认为 15 分钟。防止条件持续时事件泛滥。 |

## 在事件中捕获计算值

启用事件生成后，**事件属性**列在输出属性表格（第 3 段）中变为可用。这允许将分析表达式的计算值直接在事件创建时刻捕获到事件属性中。

例如，若分析计算窗口期间的最大电压，可以将该值同时写入元素属性（用于时序历史记录）和事件属性（与事件一起记录）。每个输出行可以独立地写入元素属性、事件属性或两者兼有。

## 与事件模板的关系

此处选择的事件模板决定：

- 事件名称和显示标签
- **事件属性**列中可用的事件属性
- 此分析生成的事件在事件标签页中的显示方式

分析生成的事件与手动创建的事件一起出现在元素的事件标签页中。它们在视觉上标注分析名称作为来源。
