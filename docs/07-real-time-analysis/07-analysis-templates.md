# 7.7 分析模板

**分析模板**将可复用的分析规则（包括触发、计算和事件生成配置）定义为[元素模板](../03-data-modeling/01-elements.md#316-元素模板)的组成部分。基于模板创建元素时，所有分析模板将自动实例化，使新元素无需任何手动配置即可获得完整的监控规则集。

## 分析模板标签页

元素模板上的**分析模板**标签页与元素上的常规分析视图工作方式类似，显示已定义的分析模板列表，列包括：**名称**、**触发类型**、**流名称**、**类别**、**状态**和**更新时间**。

可通过以下两种方式创建分析模板：

- **AI 辅助：** 在文本框中描述所需分析（例如"计算每小时最大电压"）并按回车键。IDMP 还会显示**推荐问题**供参考。
- **手动：** 在列表中点击 **+ 手动创建新分析**，自行配置触发、计算和事件生成。

工具栏包含**开/关**切换按钮，可一次性启用或禁用元素模板上的所有分析模板。

## 替换字符串

由于分析模板在多个元素间共享，对元素名称、属性或数据源的引用必须使用**替换字符串**而非硬编码值。IDMP 在每个相关输入字段提供 **+** 选择器以列出适用的替换字符串。若在元素模板上定义了自定义关键字（KEYWORD），也可在分析模板中使用，以动态引用各元素对应的数据源。

## 创建分析模板

分析模板的创建流程与普通分析基本一致，区别在于需要使用替换字符串代替硬编码的元素引用，以确保模板在不同元素间的通用性。具体步骤如下：
2. 点击**分析模板**标签页。
3. 点击 **+ 手动创建新分析**，或使用 AI 文本框描述所需分析。
4. 配置分析（触发、计算、事件生成），在需要引用元素特定值的位置使用替换字符串。
5. 点击**保存**。

每个配置步骤的完整说明请参见[创建分析](./02-creating-analysis.md)。

## 实例化行为

分析模板的核心价值在于自动实例化——当基于元素模板创建新元素时，系统自动将模板中定义的分析规则转化为该元素上可运行的具体分析，无需人工逐一配置。

基于元素模板创建元素时：

- 每个分析模板在新元素上实例化为具体的分析。
- 所有替换字符串解析为元素的实际名称、属性和数据引用。
- 分析根据配置的触发条件自动开始运行。

若元素模板启用了**允许扩展**，用户可在模板定义的分析基础上，为单个元素添加额外的自定义分析。
