**发布时间：2026-06-14**

## 亮点

1. 指标：新增一种新的属性类型，即KPI，用户可以一键创建多层流计算，汇总多层业务统计指标，并灵活构建这些指标的展示报表
1. 动作：新增一种新的模板类型，即动作，用户可以在实时分析中增加动作配置信息，当满足设置条件时触发相应的动作
1. AI 迭代升级：全新发布新版 AI 后台，支持上传SKills、配置MCP与对话风格，AI的业务分析能力大幅增强
1. 企业级安全管理：新增MFA和Passkey，支持敏感操作时对用户身份二次确认；新增企业级密码管理策略；支持在线用户与进程管理；内置离线版产品手册，以支持企业级内网部署场景。

## 特性

1. 仪表板编辑页面现支持直接添加可编辑富文本组件，便于嵌入说明或动态内容。
1. 分析功能现支持添加通知动作，可在分析触发时自动发送通知。
1. 新增 AI 文档检索 PageIndex 资源包生成功能。
1. 版本控制现已支持手动发布版本，可自主控制版本生效时机，便于审核与灰度发布。
1. 热力图现已支持除时间以外的其他维度（如分类、数值等），满足多维度数据分布分析需求。
1. 基础库新增动作模板管理功能，支持统一配置与复用。
1. AI 对话中可以直接展示 Panel 组件。
1. Excel 插件支持仅插入数据值，可自动忽略表头与说明行，简化数据写入流程。
1. 已生成的事件支持单条删除与批量删除操作，方便灵活管理事件数据。
1. CSV 导入属性数据功能现提供模板下载，便于快速准备合规数据。
1. AI Telegram 机器人集成 — 通过 Telegram Bot API Webhook 接收消息，经 ACP Agent 智能回复。支持文本/图片/文件消息，可选用户白名单控制，支持多轮对话。
1. AI 飞书机器人集成 — 通过飞书开放平台 Webhook + Event Subscription 接收消息，经 ACP Agent 智能回复。支持私聊和群聊@机器人，用户通过邀请码绑定实现身份关联，支持多轮对话上下文保持。
1. 仪表板编辑现支持面板分组，方便按类别整理与查找。
1. ACP 资源监控完整版 — 为 AI Agent 提供完整的系统资源监控能力，包括 CPU、内存、磁盘使用率实时采集与展示，支持 MCP 工具查询资源状态。
1. ACP 领域术语表 — 为 AI Agent 提供领域术语表（Domain Glossary）支持，使 Agent 能够理解并使用专业领域术语，提升对话的专业性和准确性。
1. 参考 Claude Code 实现基础 Agent，支持 ACP 协议 — 实现 Go 语言版本 AI Agent 后端，支持 ACP（Agent Communication Protocol）协议与外部 Agent 通信，提供 MCP 工具调用、会话管理、流式响应等能力。
1. 版本控制现已支持 GitHub 集成，方便与代码仓库联动管理配置变更。
1. 引入外部 Agent（Claude Code）作为 AI 服务后端 — 通过 ACP 协议将 AI 后端从内置 Python LangGraph 切换为外部 Agent，支持 MCP 工具调用、会话管理、多平台 IM 对接（飞书/Telegram/微信）。
1. 分析面板现已支持保存当前分析状态，便于随时恢复并继续分析任务。
1. 支持全量备份，可一键完整备份所有数据，保障业务可恢复性。
1. 支持 MFA/2FA 多因素认证，提升账户登录安全性。
1. 用户密码策略已升级，支持自定义复杂度与有效期，增强账户安全防护。
1. 新增质量分析主菜单，支持灵活配置菜单的显示与隐藏，方便按需启用。
1. 元素模板与事件模板现支持自定义小图标，便于快速识别与区分。
1. IDMP CLI Skills 及 ClaudeCode 插件发布。
1. 将现有 AI 功能 Skill 化 — 将 IDMP 已有的 AI 功能（面板创建、推荐、RCA、分析推荐等）封装为标准化 Skill，供 ACP Agent 按需加载调用。
1. 云服务支持 TSDB/IDMP MCP — 云服务端集成 TSDB 和 IDMP 的 MCP（Model Context Protocol）工具服务，使外部 AI Agent 可以通过 MCP 协议访问 TDengine 数据库和 IDMP 平台能力。
1. 已集成 IDMP 离线文档，支持在无网络环境下随时查阅，方便合规与参考。

## 优化

1. 管理后台系统配置的基本配置页面新增「启用质量分析菜单」开关，可灵活控制质量分析功能的入口显示。
1. 优化数据写入菜单位置，将其移至对应的 TDengine 连接节点下，使数据写入操作更贴合连接管理路径。
1. 优化属性模板详情页的刷新行为：页面刷新后仍保留在当前详情页，避免返回列表页。
1. 优化示例数据：移除非业务相关的默认 Path 属性，使示例配置更贴近实际业务场景。
1. 采用全局锁机制，保障版本控制中合并与回滚操作的原子性与数据一致性。
1. 新增资源全局锁机制，支持多用户协同操作，有效避免并发冲突。
1. AI 服务 ACP 初始化容错机制 — 将 app/__init__.py 中的 try/except 拆为两级处理：配置错误和 ImportError 导致启动失败（硬错误），MCP 初始化异常仅触发降级警告并继续运行（软降级），避免单点 MCP 故障拖垮整个 AI 服务。
1. 数据模型导入任务支持异步流程通知，可实时获取导入进度与结果。

## 修复

1. 补全 LLM API 调用的 max_completion_tokens 参数，修复私有化部署大模型交互失败的问题。
