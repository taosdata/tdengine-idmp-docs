# 8.10 根因分析

根因分析（RCA）是一项 AI 驱动的调查功能——给定一个事件，系统自动检索相关历史数据、形成关于原因的假设、验证这些假设，并生成结构化分析报告，全程无需人工干预。

该功能使用连接管理中配置的**深度思考模型**，对您的时序数据、运营知识和公开技术参考进行多步推理。

## 如何访问

从**事件详情页**访问根因分析：

1. 从顶部导航栏导航至**事件**部分。
2. 点击任意事件打开其详情视图。
3. 在详情面板右侧的工具栏中，点击**根因分析**图标（带信号波形的放大镜图标）。

根因分析面板将在事件详情页的右侧打开。

## 启动根因分析后发生什么

分析作为自动化多步工作流运行：

1. **意图识别** — 系统从事件上下文中确定分析目标。
2. **元素确认** — 从事件中提取关联元素、设备 ID、发生时间和症状描述。
3. **数据检索** — 生成 TDengine SQL 以获取该元素在周围时间窗口（通常为过去 10 天）的时序数据。
4. **数据探索** — 生成并执行 Python 分析代码，对检索到的数据进行统计探索，识别异常值、趋势和相关性。
5. **知识检索** — 通过网络搜索检索此类设备的相关技术文档或已知故障模式。
6. **假设分解** — AI 将问题分解为一个或多个关于潜在根因的子假设。
7. **假设验证** — 针对实际数据验证每个假设。
8. **报告生成** — 生成汇总所有发现的结构化 Markdown 报告。

面板实时流式展示工作流进度，您可以跟踪每个步骤的执行情况。

## 根因分析报告

最终报告是一份结构化文档，包含：

- **概述** — 事件名称、受影响系统范围、发生时间、严重程度级别
- **时间线** — 事件周围关键事件和操作的按时间顺序排列的表格
- **数据分析** — 数据探索阶段的统计发现，包括已识别的异常值或异常
- **根因假设** — AI 关于事件原因的排序假设，附有数据支持证据
- **建议措施** — 建议的纠正或预防行动

## 面板控件

| 控件 | 说明 |
|---|---|
| **刷新图标** | 为该事件重新运行根因分析 |
| **关闭（X）** | 关闭根因分析面板，返回标准事件详情视图 |

:::note
根因分析是 TDengine IDMP 1.0.14 中引入的新功能。未来版本计划增加更多根因分析入口（例如从仪表板和 AI 问答界面发起）。
:::
