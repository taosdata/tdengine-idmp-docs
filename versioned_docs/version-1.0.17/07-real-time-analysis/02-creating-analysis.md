# 7.2 创建分析

要手动创建新分析，请导航至元素的**分析**标签页，点击工具栏中的 **+** 按钮，或在空列表中点击 **+ 手动创建新分析**按钮，即可打开分析创建表单。

表单分为四个按顺序填写的编号区段：**基本信息**、**触发**、**计算**和**事件**。

## 7.2.1 基本信息

基本信息区段（分析表单第 1 段）用于定义分析的标识信息和全局行为选项，包括命名、分类以及启用策略等基础配置。

| 字段 | 说明 |
|---|---|
| **名称**（必填） | 分析的唯一名称。建议使用简洁且具有描述性的命名，例如"每小时最大电压"或"压缩机效率"。 |
| **类别** | 用于组织和筛选分析的可选类别标签，支持内联创建新标签。 |
| **创建后立即启用分析** | 勾选时（默认），分析在保存后立即开始运行；取消勾选则创建处于暂停状态的分析。 |
| **对乱序数据重新计算** | 勾选时，若接收到迟到数据（时间戳早于已处理数据），分析将对受影响的窗口重新执行计算。适用于传感器数据可能延迟或乱序到达的场景。 |
| **描述** | 可选的自由文本描述，用于说明分析的计算内容及用途。 |

## 7.2.2 触发器

触发器区段定义分析何时运行。所有八种触发类型及其参数的完整说明请参见[触发类型](./03-trigger-types.md)。

所有触发类型共享两个可选的公共字段：

- **预过滤** — 在触发评估前对数据应用的过滤表达式。仅满足过滤条件的数据行参与计算。适用于排除无效读数（例如在计算平均值前过滤掉零值）。
- **重算历史** — 启用后，分析将基于历史数据执行计算以回填输出属性。启用此字段后显示两个附加选项：
  - **优先重算历史** — 勾选时，分析在处理实时数据之前优先完成所有历史数据的计算。
  - **开始时间** — 历史数据回填的起始日期和时间。

## 7.2.3 计算

计算区段定义分析计算什么以及结果存储在哪里。完整说明请参见[计算](./04-calculation.md)。

## 7.2.4 事件

事件区段控制分析每次触发时是否生成事件。此区段默认禁用。勾选**生成**复选框以启用。完整说明请参见[生成事件](./05-generating-events.md)。

## 7.2.5 保存和放弃

点击**保存**完成分析创建。若已勾选**创建后立即启用分析**，分析将立即开始运行，并以**运行中**状态显示在列表中。

点击**放弃**取消操作。若存在未保存的更改，系统将弹出确认对话框。

:::tip
表单右侧包含可折叠的**用户指南**面板，在填写各字段时提供上下文说明。
:::
