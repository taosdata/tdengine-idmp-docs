## Step 3: `scripts/prepare.py` — Update staleness detection

Update `find_sections_needing_extraction()` to check the `content_hash` field in

```python
def find_sections_needing_extraction(
    manifest: dict,
    sections_dir: Path,
) -> tuple[list[str], list[str]]:
    """
    Returns (needs_extraction, up_to_date) lists of section_ids.

    A section needs extraction if:
      - its extraction.yaml is absent, OR
      - its extraction.yaml content_hash does not match the manifest hash
        (section content changed since last extraction)
    """
    needs = []
    up_to_date = []

    for entry in manifest.get("sections", []):
        sid = entry["section_id"]
        manifest_hash = entry["content_hash"]
        section_dir = section_id_to_dir(sid, sections_dir)
        extraction = section_dir / "extraction.yaml"

        if not extraction.exists():
            needs.append(sid)
            continue

        try:
            data = yaml.safe_load(extraction.read_text(encoding="utf-8"))
            extraction_hash = (data or {}).get("content_hash")
        except (yaml.YAMLError, OSError):
            extraction_hash = None

        if extraction_hash != manifest_hash:
            needs.append(sid)
        else:
            up_to_date.append(sid)

    return needs, up_to_date
```

Also update the existing comment in the function (line 131) which reads
"(parse.py already deleted stale extraction.yaml for changed sections.)" — remove it.

Replace the existing prompt string (lines 46–81). Two key changes:

- Agent must write `content_hash` in each `extraction.yaml` (read from `meta.yaml`)

```python
EXTRACTION_PROMPT_CONTENT = """\
# Capability Extraction Task

Read `.sections/needs-extraction.yaml`. It lists the section IDs that need
capability extraction. If `sections` is empty (`sections: []`), there is
nothing to do.

For each `section_id` in `sections`, resolve its directory path under `.sections/`:
- Split on `#`: the part before `#` is the name path; the part after is the slug.
- Replace any `/` in the name path with directory separators.
- Directory: `.sections/<name_path>/<slug>/`

Examples:
  `introduction#what-is-tdengine-idmp`  →  `.sections/introduction/what-is-tdengine-idmp/`
  `data-modeling/index#overview`        →  `.sections/data-modeling/index/overview/`

For each resolved directory:

1. Read `meta.yaml` and note the `content_hash` value.
2. Read `section.md`.
3. Identify capabilities — things the system can do: tasks, functions, or
   behaviors available to users or administrators.
   Examples: "trend-chart", "element-search", "sso-authentication".
   Do NOT treat navigation instructions, UI layout descriptions, or
   references to external systems as capabilities.
4. For each capability, classify the relation:
   - "defined" — this section explains what the capability is, how it works,
     or how to use/configure it.
   - "referenced" — this section meaningfully depends on or integrates with
     the capability, but does not define it.
   Do NOT mark incidental mentions (navigation links, see-also footnotes)
   as referenced.
5. Write `extraction.yaml` in the same directory. Include `content_hash` as
   the first field.

Output format for each `extraction.yaml`:

```yaml
content_hash: "<value from meta.yaml>"
capabilities:
  - id: "short-lowercase-hyphenated-id"
    relation: "defined"     # or "referenced"
    confidence: "high"      # or "medium" or "low"
```

If a section contains no capabilities:

```yaml
content_hash: "<value from meta.yaml>"
capabilities: []
```
"""
```

## Step 6: `scripts/prepare.py` — Automate prompt freshness

Replace the file-absence check in `write_prompt_if_needed()` with a content comparison,
so the prompt is rewritten automatically whenever `EXTRACTION_PROMPT_CONTENT` changes,
without requiring a manual file deletion or a `PROMPT_VERSION` bump:

```python
def write_prompt_if_needed(prompt_file: Path, force: bool = False) -> bool:
    """Write extraction-prompt.md if missing, stale, or force=True. Returns True if written."""
    if not force and prompt_file.exists():
        if prompt_file.read_text(encoding="utf-8") == EXTRACTION_PROMPT_CONTENT:
            return False
    write_atomic(prompt_file, EXTRACTION_PROMPT_CONTENT)
    return True
```

Key changes from the current implementation:
- Uses content comparison instead of existence check — stale prompts are updated automatically
- Uses `write_atomic()` to prevent partial writes
- Does **not** bump `PROMPT_VERSION` — content comparison replaces the version mechanism
  for the prompt file specifically

The existing `.sections/extraction-prompt.md` does not need to be manually deleted.
On the first `prepare.py` run after deployment, the content comparison will detect the
change and rewrite the file automatically.

```python
def dry_run_mode(records: list[dict], sections_dir: Path) -> int:
    """Print what parse.py would do without writing any files."""
    expected_dirs: set[Path] = {section_id_to_dir(r['section_id'], sections_dir) for r in records}

    existing_dirs: set[Path] = set()
    if sections_dir.exists():
        for p in sections_dir.rglob('section.md'):
            existing_dirs.add(p.parent)

    to_remove = sorted(d for d in existing_dirs if d not in expected_dirs)

    new_sections, updated_sections, unchanged_sections = [], [], []
    for rec in records:
        section_dir = section_id_to_dir(rec['section_id'], sections_dir)
        existing_meta = section_dir / 'meta.yaml'
        if not existing_meta.exists():
            new_sections.append(rec['section_id'])
        else:
            try:
                old_meta = yaml.safe_load(existing_meta.read_text(encoding='utf-8'))
                if old_meta.get('content_hash') != rec['content_hash']:
                    updated_sections.append(rec['section_id'])
                else:
                    unchanged_sections.append(rec['section_id'])
            except (yaml.YAMLError, OSError):
                updated_sections.append(rec['section_id'])

    print(f"[dry-run] Would write {len(new_sections)} new, {len(updated_sections)} updated, "
          f"{len(unchanged_sections)} unchanged sections")
    if new_sections:
        print(f"  NEW ({len(new_sections)}):")
        for sid in new_sections[:10]:
            print(f"    {sid}")
        if len(new_sections) > 10:
            print(f"    ... and {len(new_sections) - 10} more")
    if updated_sections:
        print(f"  UPDATED ({len(updated_sections)}):")
        for sid in updated_sections[:10]:
            print(f"    {sid}")
        if len(updated_sections) > 10:
            print(f"    ... and {len(updated_sections) - 10} more")
    if to_remove:
        print(f"  REMOVE ({len(to_remove)}) orphaned directories:")
        for d in to_remove[:10]:
            print(f"    {d.relative_to(sections_dir)}")
        if len(to_remove) > 10:
            print(f"    ... and {len(to_remove) - 10} more")
    if not (new_sections or updated_sections or to_remove):
        print("  Nothing to do — corpus is up to date.")
    print("[dry-run] No files written.")
    return 0
```

`--dry-run` is mutually exclusive with `--check`: if both are passed, print an error and exit 1.

## Step 9: `prepare.py` — CLI contract

`prepare.py` accepts four flags (`--force`, `--check`, `--dry-run`, `--bootstrap`) plus
prompt-version side effects. Their interactions are not all obvious. The table below is the
normative contract; implement guard clauses at the top of `main()` to enforce the
forbidden combinations:

| Combination | Behaviour |
|---|---|
| *(no flags)* | Normal: update prompt if stale, write `needs-extraction.yaml`, report |
| `--force` | Delete all `extraction.yaml`, then normal |
| `--check` | Read-only: exit 1 if needs > 0; writes nothing |
| `--dry-run` | Read-only: print what would happen; writes nothing; exit 0 |
| `--bootstrap` | Generate draft taxonomy; returns early; no other flags apply |
| `--force --check` | **Error** — contradictory; exit 1 with message |
| `--force --dry-run` | **Error** — contradictory; exit 1 with message |
| `--check --dry-run` | **Error** — redundant/ambiguous; exit 1 with message |
| `--bootstrap --check` | **Error** — bootstrap always writes; exit 1 with message |
| `--bootstrap --dry-run` | **Error** — exit 1 with message |
| `--bootstrap --force` | **Error** — exit 1 with message |

Writes permitted per mode:

| Mode | `needs-extraction.yaml` | `extraction-prompt.md` | `extraction.yaml` (delete) | taxonomy/aliases |
|---|---|---|---|---|
| normal | yes | if stale | no | no |
| `--force` | yes | if stale | yes (all) | no |
| `--check` | no | no | no | no |
| `--dry-run` | no | no | no | no |
| `--bootstrap` | no | no | no | yes |

Note: prompt-version side effects (deleting all `extraction.yaml` when `PROMPT_VERSION`
changes) run in normal and `--force` modes only. They do not run in `--check` or
`--dry-run` mode.

## Step 10: `scripts/validate.py` — Cross-check `extraction.yaml.content_hash`

Add a check to `validate.py` that verifies the `content_hash` field in each
`extraction.yaml` matches the corresponding `meta.yaml` for all sections present in the
section map. This catches an agent that writes a wrong hash stamp (which would be accepted
by `merge.py` but cause `prepare.py` to miss re-extraction on future content changes).

```python
def check_extraction_hash_stamps(manifest: dict, sections_dir: Path, results: Results) -> None:
    """Verify extraction.yaml.content_hash matches meta.yaml for all extracted sections."""
    for entry in manifest.get("sections", []):
        sid = entry["section_id"]
        manifest_hash = entry["content_hash"]
        section_dir = section_id_to_dir(sid, sections_dir)
        extraction = section_dir / "extraction.yaml"
        meta = section_dir / "meta.yaml"

        if not extraction.exists():
            continue  # Not yet extracted — not a validation error

        ext_data = load_yaml(extraction) or {}
        extraction_hash = ext_data.get("content_hash")

        if extraction_hash is None:
            results.warn(f"{sid}: extraction.yaml missing content_hash field")
        elif extraction_hash != manifest_hash:
            results.error(
                f"{sid}: extraction.yaml content_hash ({extraction_hash!r}) does not match "
                f"manifest ({manifest_hash!r}) — wrong hash stamp"
            )
```

Call this function from `main()` after the existing checks. This is a `validate.py`-only
addition; `merge.py` is unchanged (its path-a check against `meta.yaml` remains the
correctness gate).
